# 凯丰德官网手机端性能优化设计

日期：2026-07-21  
状态：已由用户回复“修复”，批准审查报告中的 P0/P1 优化方向  
依据：`.audit/mobile-smoothness-2026-07-21/audit.md`

## 目标

在不改变现有品牌视觉、页面结构、文案、路由和询价流程的前提下，提高 `en.kfdpack.com` 在普通安卓手机和 iPhone 上的首屏加载速度、滚动稳定性与触控响应。

## 已确认根因

1. 手机首屏附近已加载约 14.72 MiB PNG 图片。
2. Hero 图片单张约 3.00 MiB，Logo 约 1.21 MiB，实际手机显示尺寸远小于源图。
3. Hero 大图在手机上持续执行 18 秒无限缩放动画。
4. 固定导航在手机滚动时持续执行 `backdrop-filter: blur(8px)`。
5. `loading="lazy"` 仍会提前加载距离视口较近的多张 2 MiB 图片。
6. 手机页面约 6467 px 高，图片卡片和区块间距偏松，增加连续滚动距离。

## 方案比较

### 方案 A：仅调整 CSS

- 手机端关闭缩放动画、模糊和悬停效果。
- 优点：改动最小。
- 缺点：无法解决图片传输量与解码内存，是有限改善。

### 方案 B：响应式图片 + 合成层优化（采用）

- 为现有真实图片生成小、中尺寸 WebP；原 PNG 继续作为兼容回退与源文件。
- 通过 `picture`、`srcset`、`sizes` 按显示槽位分发图片。
- 手机端关闭持续缩放、背景模糊和无意义 hover 效果。
- 为离屏区块启用 `content-visibility`，并收紧手机图片高度与区块间距。
- 优点：直接解决传输、解码和滚动合成三项根因；不改变视觉方向。
- 缺点：需要新增一组派生图片资源并维护响应式引用。

### 方案 C：全站 AVIF、CDN 与预加载策略重构

- 在方案 B 基础上加入 AVIF、CDN 图片变换、资源清单和预缓存。
- 优点：理论体积最低。
- 缺点：运维复杂度高，当前小型企业展示站没有必要。

## 采用范围

### 图片资产

- 保留所有原始 PNG/JPEG。
- 首批优化以下高影响资产：
  - Header/Footer Logo
  - 首页 Hero
  - 首页工厂双图
  - 首页三张产品图
  - 首页认证缩略图
- 为图片生成与实际槽位相符的 WebP 派生文件，至少覆盖手机和桌面两个宽度档。
- Hero 使用 `fetchPriority="high"` 和 `decoding="async"`。
- 非首屏图片使用 `loading="lazy"`、`decoding="async"`。
- 提供明确的 `width`/`height` 或等效稳定比例，避免布局抖动。

### CSS 与交互

- 在 `max-width: 760px` 下：
  - Hero 图片 `animation: none`；
  - Header `backdrop-filter: none`，使用稳定深蓝背景；
  - 卡片图片高度约 260–280 px；
  - 主页区块上下间距约 52–56 px；
  - 在无 hover 设备上关闭卡片位移、缩放和重阴影。
- 保留 `prefers-reduced-motion` 规则。
- 为主页明显离屏区块使用 `content-visibility: auto` 和合理的 `contain-intrinsic-size`。

### 不在本次范围

- 不改变 Logo、颜色、字体、文案和导航结构。
- 不新增客户名称、二维码或业务数据。
- 不改变询价 Webhook 数据结构。
- 不引入 CDN、Service Worker 或新的运行时依赖。
- 不重新设计桌面页面。

## 代码边界

- 新增一个小型响应式图片组件，统一 `picture`、WebP `srcset`、回退资源和加载属性。
- `Hero`、`Header`、`Footer` 与首页图片调用该组件或使用同一数据契约。
- 图片派生文件位于 `public/assets/optimized/`，名称包含原资产语义和宽度档。
- CSS 优化集中在现有移动端媒体查询及离屏区块规则中。

## 测试与验收

### 自动化测试

1. Hero 输出 WebP `srcset`、手机尺寸候选、`fetchPriority="high"` 和 `decoding="async"`。
2. Header Logo 使用响应式资源而不是单一 2048 px PNG。
3. 非首屏首页图片保持 lazy 并使用异步解码。
4. 手机 CSS 明确关闭 Hero 动画和 Header 模糊。
5. 手机 CSS 包含离屏绘制与紧凑间距规则。
6. 现有 23 项行为测试继续通过。

### 资源验收

- 派生图片尺寸与文件体积通过脚本核验。
- 手机首屏图片目标总量约 0.3–0.6 MiB。
- 手机首屏附近图片目标总量约 1.5–3 MiB。

### 浏览器验收

- 390 × 844：首页、菜单、询价弹窗和连续滚动。
- 834 × 900：平板导航和双列布局不退化。
- 默认桌面视口：页面视觉和路由不退化。
- 所有页面无横向溢出、无破图、无控制台错误。

### 部署验收

- 测试与生产构建通过后推送 GitHub `main`。
- 在服务器生成新 release 并原子切换 `/var/www/kfd/current`。
- Nginx 配置测试通过，HTTPS 首页和五个跳转页返回正常。
- 线上资源头、页面内容与部署提交 SHA 复核一致。

## 回滚

- 原始图片始终保留，可直接恢复旧引用。
- 服务器 release 目录保留上一版本，出现异常时将 `current` 软链接切回并 reload Nginx。
