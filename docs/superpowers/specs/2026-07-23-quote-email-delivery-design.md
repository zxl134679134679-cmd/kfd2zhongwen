# 官网询价自动邮件设计

## 目标

客户在凯丰德官网完成三步询价表单后，网站把客户资料提交到同一域名下的轻量邮件接口。服务器使用 163 邮箱 SMTP 将资料发送到 `zxl134679@163.com`。只有邮件服务确认发送成功，网页才显示成功状态。

## 方案选择

使用现有美国服务器提供同源接口 `POST /api/quote`，不在该服务器上安装 n8n。

选择原因：

- 当前服务器内存较小，轻量接口比常驻 n8n 服务占用更低。
- 接口和网页使用同一个 `en.kfdpack.com` 域名，不需要新增子域名和跨域配置。
- 邮箱授权码只存在服务器环境文件中，不进入浏览器代码、构建产物或 GitHub。
- 现有 React 表单已经具备结构化提交逻辑，只需收紧真实成功和失败反馈。

## 系统结构

### 网页表单

`QuoteDialog` 继续保留现有三步设计和字段：

- 产品类型
- 尺寸
- 数量
- 材质 / 楞型
- 印刷需求
- 交付地区
- 补充说明
- 客户联系方式
- 页面地址、语言和提交时间

生产环境提交地址固定为 `/api/quote`。本地开发通过显式配置测试地址；未配置可用接口时不得显示“需求已发送”。

### 邮件接口

服务器运行一个小型 Node.js 服务，仅提供 `POST /api/quote`：

1. 限制请求体大小并只接受 JSON。
2. 校验来源标记和必填字段。
3. 清理客户输入，避免把未处理的 HTML 放入邮件。
4. 收件人固定为 `zxl134679@163.com`，不采信浏览器传入的收件地址。
5. 使用服务器环境变量中的 163 SMTP 账号和授权码发送邮件。
6. 成功返回 `2xx`；校验失败、频率过高或邮件发送失败返回明确的非 `2xx` 状态。

接口不保存客户资料到磁盘或数据库。服务器日志只记录时间、状态和匿名请求标识，不记录完整联系方式、备注或邮箱授权码。

### Nginx

现有 Nginx 继续负责网站静态文件和 HTTPS，并新增：

- `/api/quote` 反向代理到本机邮件服务。
- 限制请求方法和请求体大小。
- 基础频率限制，减少自动化垃圾提交。
- 保留真实客户端地址供频率限制使用。

### 邮件内容

发件人与收件人均使用 `zxl134679@163.com`。主题格式：

`【凯丰德官网新询价】产品类型 - 客户联系方式`

正文采用清晰的 HTML 表格，并附带纯文本版本。正文展示全部表单字段、提交页面、语言和服务器接收时间。缺少的非必填字段统一显示“未填写”。

## 数据流

1. 客户完成表单并点击“提交需求”。
2. 网页将结构化 JSON 发送到 `/api/quote`，提交按钮进入“提交中”状态。
3. Nginx 将请求转发给本机邮件服务。
4. 邮件服务校验内容并通过 163 SMTP 发信。
5. SMTP 成功后接口返回成功，网页显示“需求已发送”。
6. 任一步失败时接口返回错误，网页恢复可编辑状态并显示备用联系方式。

## 安全和防滥用

- 163 邮箱客户端授权码只由网站所有者填写到服务器环境文件。
- 环境文件权限只允许服务账户读取，并明确加入忽略规则。
- 前端不再发送或决定 `recipientEmail`。
- 服务端固定收件人、字段长度和允许的数据类型。
- 增加隐藏的机器人诱饵字段；该字段被填写时拒绝请求。
- Nginx 和服务端共同限制提交频率。
- 错误响应不暴露 SMTP 主机、账号、授权码或内部堆栈。

## 网页反馈

- 提交成功文案修改为“需求已发送”，说明业务人员将尽快联系。
- 邮件接口不可用或发送失败时显示：“提交失败，请稍后重试，或通过邮箱联系我们。”
- 不再允许“没有配置接口也显示成功”的演示回退。
- 保留键盘焦点、移动端滚动锁定和现有对话框无障碍行为。

## 部署方式

- 邮件服务以系统服务运行，异常退出后自动重启。
- SMTP 配置通过服务器环境文件注入。
- Nginx 配置修改前先备份并执行配置检查。
- 前端、本地服务和 Nginx 都验证通过后再切换生产配置。
- GitHub 只保存源代码、示例环境文件和部署说明，不保存真实授权码。

## 测试与验收

### 自动测试

- 网页必须向 `/api/quote` 发送规定字段。
- 接口未配置或返回失败时，网页不得显示成功。
- 接口成功时显示“需求已发送”。
- 服务端拒绝缺少必填字段、过长内容、错误类型和机器人诱饵字段。
- 服务端忽略浏览器传入的收件人。
- 邮件正文对客户输入进行转义。

### 部署验证

- 前端测试和生产构建通过。
- 邮件服务单元测试通过并能由系统服务启动。
- Nginx 配置检查通过。
- 从生产网页提交带唯一测试编号的询价。
- `zxl134679@163.com` 收到字段完整的测试邮件。
- 服务器日志中没有邮箱授权码和完整客户资料。
- 手机和电脑上分别验证提交中、成功和失败状态。

## 实施所需的用户操作

网站所有者需要登录 163 邮箱，开启 SMTP 服务并生成客户端授权码。授权码由网站所有者直接填写到服务器配置界面或终端，不通过聊天发送，也不写入项目仓库。
